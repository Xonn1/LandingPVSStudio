Выявляет не только опечатки, мёртвый код и другие ошибки, но и потенциальные уязвимости (Static Application Security Testing).

Классификация предупреждений

Для выстраивания процесса безопасной разработки по ГОСТ Р 56939—2024 необходимо проводить регулярный поиск уязвимостей на этапе эксплуатации жизненного цикла ПО.

Инструментальное средство PVS-Studio разрабатывается с учётом требований ГОСТ Р 71207–2024 и может использоваться при разработке безопасного ПО.

Применяется в сертификационных лабораториях, аккредитованных в системах сертификации ФСТЭК России.

В 2025 году PVS-Studio участвует в испытаниях ФСТЭК для выработки единой методики соответствия требованиям ГОСТ.

Стандарт OWASP ASVS — список требований к безопасности приложений и тестов.

Классификация предупреждений PVS-Studio согласно OWASP

Common Weakness Enumeration — систематизированный список типичных уязвимостей в ПО.

PVS-Studio связывает свои предупреждения с CWE, упрощая интерпретацию анализа.

SEI CERT Coding Standards — устранение распространённых уязвимостей в коде.

PVS-Studio проверяет соответствие этим стандартам для языков C, C++, Java.
